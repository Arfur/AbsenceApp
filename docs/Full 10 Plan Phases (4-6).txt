PHASE 4 — TABLE SYSTEM (BLAZOR VERSION)
Goal:
  - Introduce a unified, reusable, configurable table system built entirely with Blazor components.
  - Support sorting, filtering, searching, pagination, inline actions, skeleton loading, and empty states.
  - Use JSON-driven configuration (from Phase 2).
  - Ensure all new table components are additive and do not break existing pages.
  - Prepare for gradual migration of existing table pages.

Constraints:
  - Do NOT modify or remove existing table implementations.
  - Do NOT break existing pages.
  - All new table components must be isolated under a new namespace.
  - All styling must use CSS variables and scoped .razor.css files.
  - All configuration must come from table-schema.json.

----------------------------------------------------------------------
4.1 TABLE COMPONENT ARCHITECTURE
----------------------------------------------------------------------

4.1.1 Create folder for table components
  - Path: /Components/TableV2/

4.1.2 Create TableV2.razor
  - Responsibilities:
      - Render table header
      - Render filter row (optional)
      - Render data rows
      - Render inline actions column
      - Render pagination
      - Render skeleton rows when loading
      - Render empty state when no data
  - Parameters:
      - Items (IEnumerable<T>)
      - Columns (List<TableColumnModel>)
      - IsLoading (bool)
      - PageIndex
      - PageSize
      - TotalCount
      - OnPageChanged
      - OnSortChanged
      - OnFilterChanged
      - OnSearchChanged
      - OnView
      - OnEdit

4.1.3 Create TableV2.razor.css
  - Style using CSS variables:
      --table-header-bg
      --table-row-bg
      --table-border-color
      --space-*
      --radius-*
  - Use CSS Grid or table layout.

4.1.4 Create TableColumnModel.cs
  - Path: /Models/TableV2/TableColumnModel.cs
  - Properties:
      - string ColumnName
      - string DisplayName
      - bool Visible
      - bool Sortable
      - bool Filterable
      - bool Searchable
      - string Alignment
      - int Order

4.1.5 Create TableConfigService.cs
  - Path: /Services/TableV2/TableConfigService.cs
  - Responsibilities:
      - Load table-schema.json from wwwroot/config/designsystem/
      - Provide column definitions for each table
      - Cache results

4.1.6 Register service in MauiProgram.cs
  - builder.Services.AddSingleton<TableConfigService>();

4.2 COLUMN CONFIGURATION (VISIBLE, SORTABLE, FILTERABLE, SEARCHABLE)
----------------------------------------------------------------------

4.2.1 Implement column visibility
  - TableV2.razor:
      - Only render columns where Visible = true.

4.2.2 Implement sorting
  - Clicking header triggers:
      OnSortChanged(columnName, direction)
  - Sorting direction cycles:
      None → Asc → Desc → None

4.2.3 Implement filtering
  - If Filterable = true:
      - Render input in filter row
      - Trigger OnFilterChanged(columnName, value)

4.2.4 Implement searching
  - Search bar is external to table
  - TableV2 receives search text via parameter
  - Only applies to Searchable columns

4.2.5 Implement alignment
  - Apply CSS class:
      text-left
      text-center
      text-right

4.2.6 Implement column ordering
  - Sort columns by Order property before rendering.

PHASE 6 — DATA & SERVICES (BLAZOR VERSION)
Goal:
  - Establish a unified, scalable, maintainable data and service layer for the Absence MAUI Blazor Hybrid app.
  - Introduce consistent API service patterns, data models, ViewModels, and state management.
  - Ensure all new data/service components are additive and do not break existing functionality.
  - Prepare for gradual migration of existing pages to the new data/service architecture.

Constraints:
  - Do NOT modify or remove existing services.
  - Do NOT break existing API calls or data flows.
  - All new services must be introduced in parallel under a new namespace.
  - All new models must be additive.
  - All new ViewModels must not replace existing ViewModels.
  - All changes must be additive and reversible.

6.1 API SERVICE ARCHITECTURE
----------------------------------------------------------------------

6.1.1 Create folder for new API services
  - Path: /Services/ApiV2/

6.1.2 Create ApiClientV2.cs

  - Responsibilities:
      - Wrap HttpClient
      - Provide:
          Task<T> GetAsync<T>(string endpoint)
          Task<T> PostAsync<T>(string endpoint, object body)
          Task<T> PutAsync<T>(string endpoint, object body)
          Task DeleteAsync(string endpoint)
      - Handle:
          - Base URL (from appsettings.json)
          - Authentication headers (if required)
          - Error handling
          - Logging (optional)
  - Must NOT replace existing HttpClient usage.

6.1.3 Create ApiServiceBaseV2.cs
  - File: /Services/ApiV2/ApiServiceBaseV2.cs
  - Responsibilities:
      - Provide protected ApiClientV2 instance

      ClassesApiServiceV2.cs
