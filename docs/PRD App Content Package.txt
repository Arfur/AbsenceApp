# 1. **Project Overview**

The Absence Application is a school‑focused management system designed to track:

- Staff absences  
- Student absences  
- Attendance registers  
- Medical information  
- Safeguarding flags  
- Staff roles, assignments, and responsibilities  
- Devices issued to staff  
- External accounts  
- Audit logs for all critical actions  

The system is being developed as a **Blazor Hybrid** application using **.NET** and **Entity Framework Core**, with a **local SQL Server Express** database during development.  
When deployed, the database will be migrated to the school’s **on‑premises Microsoft SQL Server** instance.

The application is intended for:

- School administrators  
- Senior leadership  
- Teachers  
- Office staff  
- Attendance officers  
- Safeguarding leads  

It supports multi‑school setups, year groups, phases, classes, and detailed audit trails.

---

# 2. **Technical Architecture Summary**

### **Frontend**
- Blazor Hybrid (MAUI + Blazor WebView)
- C# Razor components
- Planned migration to MudBlazor for modern UI components

### **Backend**
- .NET 8 (or latest LTS)
- Entity Framework Core for ORM
- Repository + service layer pattern (planned)

### **Database**
- SQL Server Express (development)
- SQL Server Standard/Enterprise (production)
- EF Core migrations for schema evolution

### **Deployment**
- Local development: SQL Express instance  
- Production: School server SQL instance  
- Migration via EF Core + SQL backup/restore

---

# 3. **Database Philosophy**

The schema is designed around:

- **Normalised relational structure**  
- **Clear foreign key relationships**  
- **Auditability** (every major action is logged)  
- **Multi‑school support**  
- **Role‑based access**  
- **Extensibility** (new absence types, roles, responsibilities, etc.)  

The system separates:

- **Users** (accounts)  
- **Staff** (HR‑style records)  
- **Students** (MIS‑style records)  
- **Assignments** (roles, classes, responsibilities)  
- **Absences** (staff + student)  
- **Attendance** (registers + marks)  
- **Audit logs** (for every change)
