# **1. Overview**

The V2 Layout System is a **fully modular, isolated, token‑driven UI framework** that replaces the V1 layout **without modifying any V1 files**.

It provides:

- A new Sidebar  
- A new Header  
- A new Breadcrumb  
- A new ScrollSpy system  
- A new Page Template system  
- Full design‑system token integration  
- Smooth transitions and animations  
- Responsive behaviour  
- A consistent layout foundation for all V2 pages

All V2 layout components live under:

```
AbsenceApp.Client/Components/LayoutV2/
```

# **2. V2 Layout Components**

## **2.1 SidebarV2**
### **Purpose**
- Primary navigation container  
- Collapsible/expandable  
- Token‑driven styling  
- Dynamic menu loading  

### **Key Behaviours**
- Smooth animated collapse/expand  
- Width transitions without layout jitter  
- Menu items loaded from DB via `ui_MenuItems` + `RoleMenuItems`  
- Active route highlighting  
- Scrollable content area  
- Token‑driven colours, spacing, borders  

### **State**
- `IsCollapsed` persisted in local storage  
- Sidebar responds instantly after login (NF‑06)

---

## **2.2 HeaderV2**
### **Purpose**
- Top navigation bar  
- Houses theme toggle, notifications, user context  

### **Key Behaviours**
- Token‑driven height, spacing, colours  
- Notification icon reflects persistent absence alerts  
- Theme toggle updates CSS variables live  
- Breadcrumb is rendered inside HeaderV2  

---

## **2.3 BreadcrumbV2**
### **Purpose**
- Replaces page titles entirely (F‑04)  
- Reflects current route  
- Token‑driven spacing and typography  

### **Key Behaviours**
- Dynamically generated from route metadata  
- Supports nested routes  
- No hardcoded titles  
- Always visible  

---

## **2.4 ScrollSpyV2**
### **Purpose**
- Tracks which section of a page is visible  
- Updates sidebar or local navigation highlights  

### **Key Behaviours**
- IntersectionObserver‑based  
- Smooth highlight transitions  
- Token‑driven highlight colours  
- Works with PageTemplateV2 section anchors  

---

## **2.5 PageTemplateV2**
### **Purpose**
- Standardised layout wrapper for all V2 pages  
- Ensures consistent spacing, typography, and structure  

### **Key Behaviours**
- Token‑driven spacing (16px grid)  
- Responsive 1–3 column form layout  
- Sticky header support  
- ScrollSpy section anchors  
- Built‑in loading and empty states

# **3. V2 Layout Architecture Rules**

### **3.1 V1 and V2 must coexist**
- V1 layout is frozen  
- V2 layout must not break V1  
- V2 pages must render correctly even if incomplete  

### **3.2 V2 layout must be token‑driven**
- No hardcoded colours  
- No hardcoded spacing  
- No hardcoded borders  
- All styling must come from CSS variables generated by the Token System  

### **3.3 V2 layout must be self‑contained**
- No modifications to V1 CSS  
- No modifications to V1 components  
- No global overrides  

### **3.4 V2 layout must support dynamic menu loading**
- SidebarV2 must load menu items from DB  
- No hardcoded menu structures  

---

# **4. Integration with Token System**

All V2 layout components must:

- Load tokens from `components.json`  
- Use CSS variables for all styling  
- Support dynamic updates when tokens change  
- Use the `DesignSystemConfigService` for config loading  

Examples:

- Sidebar background → `--ds-sidebar-bg`  
- Header height → `--ds-header-height`  
- Breadcrumb text → `--ds-breadcrumb-text`  

---

# **5. Integration with Menu System V2**

SidebarV2 must:

- Load menu items from `ui_MenuItems`  
- Filter them using `RoleMenuItems`  
- Prune empty parent categories  
- Render final navigation tree  

No hardcoded menu items are allowed.

---

# **6. Integration with PageTemplateV2**

PageTemplateV2 provides:

- Standardised spacing  
- Section anchors for ScrollSpy  
- Token‑driven typography  
- Responsive grid layout  

All V2 pages must use PageTemplateV2.
