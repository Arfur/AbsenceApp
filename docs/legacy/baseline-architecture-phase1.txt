────────────────────────────────────────────────────────────────────────────────
1. ARCHITECTURE OVERVIEW
────────────────────────────────────────────────────────────────────────────────

Technology Stack:
  Runtime        : .NET 8.0 MAUI Blazor Hybrid (Windows desktop)
  UI Engine      : Blazor WebView — Razor components rendered in MAUI WebView
  Styling        : CSS custom properties + Bootstrap 5 (no XAML styles in UI)
  Data Access    : Entity Framework Core — LocalDB (SQL Server LocalDB)
  API Layer      : ASP.NET Core Minimal API (separate process, optional)
  DI Container   : Microsoft.Extensions.DependencyInjection (MauiProgram.cs)
  Auth           : Simple username/password (AuthService, AppStateService)
  State          : AppStateService singleton (in-memory, no external store)
  Testing        : xUnit (server) + bUnit (client components)

Rendering Model:
  - MAUI shell hosts a single BlazorWebView (MainPage.xaml)
  - All UI rendering happens inside the WebView
  - CSS from wwwroot/css/ is loaded via index.html <link> tags
  - Blazor scoped CSS (.razor.css) is compiled and bundled automatically
  - No native MAUI XAML controls are used in the UI layer

5. ARCHITECTURAL RULES — V2 MUST FOLLOW
────────────────────────────────────────────────────────────────────────────────

RULE 1  : All V2 components use CSS custom properties (tokens from Phase 2).
          No hard-coded color, spacing, or font values in .razor.css files.

RULE 2  : All V2 Razor components are self-contained with co-located .razor.css
          scoped style files.

RULE 3  : All V2 services follow the DI pattern. Services must be registered
          in MauiProgram.cs before a V2 page can use them. (MauiProgram.cs
          changes require explicit user instruction per phase.)

RULE 4  : All V2 JSON configuration files live under wwwroot/config/ and are
          loaded asynchronously via HttpClient or JSInterop.

RULE 5  : No V2 component imports or inherits from V1 layout components.
          V2 layout is self-contained.

RULE 6  : V2 module pages (Phase 7) use only V2 templates, V2 tables, V2
          services, and V2 ViewModels. Zero dependency on V1 pages.

RULE 7  : Phase 9 (FrameworkV2) contains COPIES, not references, of V2 assets.
          No cross-project reference is introduced.

RULE 8  : V1 app.css CSS variable names are reused in V2 token files to ensure
          V2 components visually match V1 until theming (Phase 8) is applied.

RULE 9  : All V2 files include the standard AbsenceApp file header comment:
            // File    : [filename]
            // Project : AbsenceApp.Client
            // Phase   : V2 Design System — Phase [N]
            // Created : [date]

RULE 10 : Stop and ask the user before registering any V2 service in
          MauiProgram.cs. This is the only V1 file that may be modified,
          and only with explicit user approval.
